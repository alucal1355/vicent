3. Gestión de accesos. Sistemas NAC

Actualmente muchas empresas están utilizando el teletrabajo como una de las soluciones para permitir que varias personas operen desde el exterior de la empresa, como si estuvieran en ella, es con una red privada virtual o VPN. El sistema operativo en red Windows Server 2019 incluye este rol dentro del listado de funciones que pueden añadirse como servicio en el sistema. Como solución, se puede realizar una implementación simple de VPN utilizando un único servidor, pero en este escenario de estudio, se completa el uso de esta tecnología con un servidor RADIUS (Remote Authentication Dial-In User Service).

VPN es una tecnología de transporte de datos a la que se complementa de tecnologías de cifrado y autenticación. Con RADIUS realizamos esta última función.

Un caso práctico de autenticación RADIUS, sería la autenticación a una red wifi con WPA2-ENTERPRISE en el cual no necesitamos una clave secreta compartida para acceder a ella. En lugar de una contraseña, utilizaríamos un usuario para autenticar.

Nosotros vamos a poner esto en práctica mediante la creación de un esquema de 3 máquinas en la que una actúa como servidor del dominio y servidor RADIUS, otra como cliente RADIUS y proveedor de acceso VPN al exterior y finalmente uno o varios equipos que conectan desde el exterior para hacer uso de la red corporativa.

Trabajo que se va a realizar:

  1. Instalación/configuración el Servidor Radius en un Servidor Windows 2019 promocionado a CD.
  2. Instalación y configuración del cliente Radius  y Servidor VPN en un Servidor Windows 2019.
  3. Configuración de un Equipo con Windows 10 Enterprise como cliente VPN y autenticación RADIUS.
  4. Pruebas de AAA con el cliente Windows 10 Enterprise.
