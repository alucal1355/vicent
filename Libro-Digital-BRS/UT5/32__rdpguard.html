3.2 RdpGuard

1. Introducción

En este capítulo, se describe cómo configurar una herramienta para la prevención de intrusiones en entornos Windows. 

En este caso se usa RdpGuard que protege los servidores de ataques de fuerza bruta en distintos protocolos y servicios, para este caso, se verá RDP o FTP.

Esta aplicación supervisa los intentos de acceso para detectar intentos fallidos de inicio de sesión, detectando las direcciones IP atacantes para bloquearlas o crear listas negras.

1.1 Recursos

  • Equipo Windows 10 Enterprise conectado en una red pública 192.168.0.0 - Equipo Objetivo.
  • Equipo kali Linux conectado en una red pública 192.168.0.0 - Equipo Atacante.

5. Ataque de fuerza bruta

Mediante la herramienta hydra se realiza un ataque de fuerza bruta con el diccionario rockyou.

En primer lugar, hay que descargar el diccionario con wget.

┌──(root💀kali)-[~]
└─# wget https://github.com/brannondorsey/naive-hashcat/releases/download/data/rockyou.txt

Figura 6. Descarga del diccionario rockyou.

Una vez descargado el diccionario, ya se puede realizar el ataque, que consiste en el puerto RDP (escritorio remoto).

Figura 7. Número de Puerto RDP configurado en el equipo objetivo.

┌──(root💀kali)-[~]
└─# hydra -L rockyou.txt -p rockyou.txt rdp://192.168.0.33:3389

Figura 8. Ejecución del comando hydra.

donde los parámetros que se le pueden pasar son:

  • -l: Se indica el usaurio del servicio.
  • -L: Se puede utilizar una lista de nombres de usuario.
  • -p: Se indica que se utilizará una sola palabra como password.
  • -P: Se puede utilizar una lista de passwords.
  • -v o -V: Verbose-
  • -R: Permite restaurar la sesión de la última vez que la herramienta fue utilizada (útil para no perder el estado del ataque).

5.1 Resultado del ataque

Tras realizar el ataque a la máquina objetivo Windows, el RdpGuard detecta los intentos de conexión al protocolo RDP y bloquea la máquina atacante.

Figura 9. Bloqueo del ataque.

6. RdpGuard - FTP

A continuación, se va a comprobar que también bloquea la dirección mediante el protocolo FTP. Para ello, hay que desbloquear la dirección que se ha bloqueado anteriormente, ya que si no, no se podrá comunicar con el equipo.

Figura 10. Desbloqueo del ataque.

A continuación, se habilita la protección FTP y se selecciona el método de monitorización de WinPcap, para ello, hay que pulsar sobre 'configurar...' y en la ventana nueva, hay que seleccionar nuestra tarjeta de red.

Figura 11. Habilitación de la protección FTP y se selección del método de monitorización de WinPcap.

Una vez realizado esto, hay que guardar y ya se podrá ver como el servicio FTP tiene la protección habilitada.

Figura 12. Protección FTP habilitada.

El siguiente paso es abrir un servidor FTP para tener la conexión disponible en nuestro equipo y poder atacarlo desde la otra máquina.

Si se quiere abrir un servidor FTP en nuestro equipo, lo primero que se tiene que hacer es activar la característica de Windows de IIS y Servidor FTP.

Figura 13. Activar la característica de Windows de IIS y Servidor FTP.

A continuación, desde el panel de control se puede acceder al panel de Administrador de IIS.

Figura 14. Acceso al panel Administrador de IIS.
