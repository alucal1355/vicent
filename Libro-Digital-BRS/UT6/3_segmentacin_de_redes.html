3. Segmentación de redes

1. Introducción

La segmentación de red es una práctica que consiste en dividir las redes en diferentes «zonas de seguridad» o subredes de forma lógica para poder conectar más equipos sin que esto conlleve a una pérdida de rendimiento y recursos.

También se utiliza por motivos de seguridad, ya que si algún equipo se ve comprometido por algún ataque malicioso, solo se comprometería el segmento de red al que pertenece y no toda la red.

El proceso de segmentación de red implica dividir una red física en diferentes subredes lógicas. Una vez la red se ha subdividido en unidades más pequeñas y manejables,  se aplican controles a los segmentos individuales compartimentados.

Problemas presentados por IP Versión 4

Debido a su formato IPv4 comenzó a presentar problemas de agotamiento de direcciones IP, por lo tanto se fueron sucediendo distintos tipos de soluciones para alargar la vida útil de IPv4 entre estas, las que ya hemos comentado como:

  • división en subredes.
  • enrutamiento sin clases, introducción del concepto CIDR.
  • Uso de NAT y direcciones IP privadas.

De forma posterior nació VLSM (Variable Length Subnet Mask). Máscaras de subred de tamaño variable. Otra de las funciones de VLSM es descentralizar las redes y de esta forma conseguir redes más seguras y jerárquicas.

2. Buenas prácticas para la segmentación de redes

Esta arquitectura es muy segura cuando se configura correctamente; pero si no se hace bien, puede traer consecuencias adversas para la organización.

A continuación, se muestran algunas buenas prácticas que se pueden seguir:

  • Utiliza infraestructuras PKI, WPA2/WPA2/infraestructura Enterprise y RADIUS/TACACS+: Esto proporciona un depósito central de usuarios o dispositivos autorizados para acceder a la red al utilizar certificados para autenticar el servidor y el dispositivo.
  • Crea y configura segmentos VLAN. Una VLAN  permitirá agrupar periféricos.
  • Procura que cada segmento tenga su propia configuración de dirección IP, así como su modo de enrutamiento, control de acceso e interfaces (grupos WiFi SSID, Ethernet y VLAN).
