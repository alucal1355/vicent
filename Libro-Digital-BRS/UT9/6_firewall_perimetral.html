6. Firewall perimetral

1. Introducción

Las dos formas básicas de diseñar una DMZ son el modelo con un cortafuego (también conocido como modelo de tres patas, en inglés three legged model) y el modelo dual o con dos cortafuegos (también conocido como modelo de espalda con espalda, en inglés back-to-back model).

Estas arquitecturas se pueden combinar y ampliar para crear arquitecturas mucho más complejas en función de las necesidades de la red.

En el modelo con un cortafuego se configura una interfaz para la red externa, otra para la red interna y una tercera para la DMZ.

Figura 1. Modelo único firewall.

El dispositivo que actúa como cortafuego debe tener capacidad suficiente para gestionar el tráfico entre las tres redes. Además, es un punto único de fallo.

El modelo dual minimiza en parte estas debilidades y resulta un diseño más seguro, aunque resulta más costoso tanto económicamente como de gestionar.

Figura 2. Modelo dual de firewall.

El primer cortafuego, conocido como front-end firewall o cortafuego de perímetro (en la figura dmz externa), se configura para permitir solo el tráfico destinado a la DMZ.

El segundo cortafuego, conocido como back-end firewall o cortafuego interno, se configura para permitir solo el tráfico desde la red interna a la DMZ. A partir de esta aproximación básica se incluyen las reglas de filtrado necesarias para adaptarse a los requisitos de la red.

La seguridad del modelo dual se incrementa si los dos cortafuegos son de diferentes fabricantes. Una posible vulnerabilidad que pueda ser explotada por un atacante malicioso en uno de ellos es poco probable que también esté presente en el otro cortafuego.

Tener un cortafuego perimetral de distinto fabricante que el cortafuego interno se puede considerar parte de la estrategia de seguridad de defensa en profundidad.
