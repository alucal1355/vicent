2. Sistemas de prevención y protección frente a intrusiones

1. Introducción

Un IDS (Intrusion Detection System, -sistema de detección de intrusos-) es un sistema informático que detecta los accesos no autorizados a una red (NIDS) o a un equipo (HIDS). Suele estar compuesto de dos elementos: una sonda (o sensor) encargada de detectar las anomalías y un sistema de almacenamiento. Una vez almacenados los datos se puede enviar una alerta para que el personal técnico evalúe si se trata de un posible ataque o de una falsa alarma.

Un IPS (Intrusion Prevention System- sistema de prevención de intrusos) es un sistema con capacidad para tomar decisiones que afectan al control acceso a la red o a los equipos en función del tráfico detectado. Frente al comportamiento pasivo de un IDS, el IPS tiene un comportamiento activo.

El tratamiento de los registros generados se realiza con la ayuda de un SIEM (Security Information and Event Management-sistema de gestión de eventos e información de seguridad). Las herramientas SIEM permiten unificar y centralizar los registros y así recopilar, normalizar y correlacionar eventos de seguridad, proporcionando herramientas de detección de amenazas y una cierta inteligencia que facilitan al personal técnico la evaluación del impacto de un ataque o el descarte de falsos positivos.

2. ¿Qué es un log?

Un log es un registro que deja un sistema informático. Por ejemplo: accesos de usuarios, actividades de borrado y cambios realizados en el sistema. 

¿Cuál es la Utilidad de los logs?

Con esta información se puede ver claramente las actividades que se han realizado en los sistemas y, por lo tanto, con un pequeño análisis se podrían detectar situaciones extrañas y anómalas. Este análisis podría funcionar en entornos pequeños con pocos sistemas, pero ¿qué pasa cuando se tienen múltiples equipos escribiendo y registrando logs?. El análisis de cada log se haría imposible. No se tendría la capacidad para analizar todos estos sistemas.

Es en este momento donde entran en juego los sistemas IDS, IPS, HIDS, NIPS, SIEM.

Información importante de los logs

  • Inicios de sesión.
  • Errores de autenticación.
  • Cambio de privilegios.
  • Creación de usuarios.
  • Modificación de grupos.
  • Modificación de Políticas.
  • Ejecución de comandos por parte de administradores.
  • Equipos contaminados.
  • Archivos de sistema.

Formatos de los log

  • Formato tiempo.
  • Identificadores únicos.
  • Formato texto.
  • Json o XML.

Common Event Format

Es un estándar para la interoperabilidad de dispositivos y aplicaciones que generan eventos o registros. El estándar define una sintaxis para los registros de log. 

Se compone de un prefijo estándar y una extensión variable que se formatea como pares clave - valor.
