11.3 Creación de cuentas LDAP

1. Creación de cuentas LDAP

Antes de crear las cuentas de OpenLDAP necesitamos definir unidades organizativas donde almacenar los usuarios y grupos. Para este caso de ejemplo, se han añadido las uo usuarios destinada a almacenar los usuarios del sistema y grupos para almacenar los grupos del sistema.

root@orion:/home/administrador# nano /etc/ssl/openldap/users-ou.ldif

Y añadimos las siguientes líneas:

dn: ou=usuarios,dc=ciber,dc=local
objectClass: organizationalUnit
objectClass: top
ou: usuarios

dn: ou=grupos,dc=ciber,dc=local
objectClass: organizationalUnit
objectClass: top
ou: grupos

Guardamos con CTRL + O y salimos con CTRL + X.

Antes de actualizar la base de datos con las unidades organizativas necesitamos ajustar ciertos controles de acceso. Para ello modificaremos el siguiente archivo:

root@orion:/home/administrador# nano /etc/ssl/openldap/update-mdb-acl.ldif

Y añadimos las líneas siguientes:

dn: olcDatabase={1}mdb,cn=config
changetype: modify
replace: olcAccess
olcAccess: to attrs=userPassword,shadowLastChange,shadowExpire
  by self write
  by anonymous auth
  by dn.subtree="gidNumber=0+uidNumber=0,cn=peercred,cn=external,cn=auth" manage
  by dn.exact="cn=readonly,ou=usuarios,dc=ciber,dc=local" read
  by * none
olcAccess: to dn.exact="cn=readonly,ou=usuarios,dc=ciber,dc=local"  by dn.subtree="gidNumber=0+uidNumber=0,cn=peercred,cn=external,cn=auth" manage by * none
olcAccess: to dn.subtree="dc=ciber,dc=local" by dn.subtree="gidNumber=0+uidNumber=0,cn=peercred,cn=external,cn=auth" manage
 by users read
 by * none

Guardamos con CTRL + O y salimos con CTRL + X.

Para consolidar los cambios realizados, hay que guardar y actualizar la base de datos de ACLs. El comando es el siguiente:

root@orion:/home/administrador# ldapadd -Y EXTERNAL -H ldapi:/// -f /etc/ssl/openldap/update-mdb-acl.ldif

A continuación, se crean las unidades organizativas que hemos creado anteriormente en el fichero users-ou.ldif. El comando es el siguiente:

root@orion:/home/administrador# ldapadd -Y EXTERNAL -H ldapi:/// -f /etc/ssl/openldap/users-ou.ldif

Con todo esto, ya tendremos creado la configuración necesaria. Ahora vamos a crear un primer usuario, para ello, hay que crear un nuevo archivo donde definiremos el usuario.

root@orion:/home/administrador#nano usuario1ldap.ldif

Añadimos las siguientes líneas:

dn: uid=usuario1ldap,ou=usuarios,dc=ciber,dc=local
objectClass: inetOrgPerson
objectClass: posixAccount
objectClass: shadowAccount
uid: usuario1ldap
cn: usuario1
sn: ldap1
loginShell: /bin/bash
uidNumber: 10000
gidNumber: 10000
homeDirectory: /home/usuario1ldap
shadowMax: 60
shadowMin: 1
shadowWarning: 7
shadowInactive: 7
shadowLastChange: 0

dn: cn=usuario1ldap,ou=grupos,dc=ciber,dc=local
objectClass: posixGroup
cn: usuario1ldap
gidNumber: 10000
memberUid: usuario1ldap

Quedando de la siguiente forma:

Guardamos con CTRL + O y salimos con CTRL + X.

A continuación, hay que crear el usuario que se ha definido en el fichero anterior mediante el siguiente comando:

root@orion:/home/administrador# ldapadd -Y EXTERNAL -H ldapi:/// -f usuario1ldap.ldif

Definición de contraseña para usuario LDAP

Para definir la contraseña sobre un usuario ya creado (en concreto usuario1ldap creado anteriormente), el comando a utilizar es el siguiente:

root@orion:/home/administrador# ldappasswd -H ldapi:/// -Y EXTERNAL -S "uid=usuario1ldap,ou=usuarios,dc=ciber,dc=local"

Para verificar la operación anterior, podemos utilizar el siguiente comando:

root@orion:/home/administrador# ldapwhoami -h ciber.local -x -D "uid=usuario1ldap,ou=usuarios,dc=ciber,dc=local" -W

2. BIND DN. Usuario por defecto para consultas al directorio LDAP

Mediante este proceso, permitiremos que los equipos busquen y consulten el directorio para realizar la autenticación de usuarios en su equipo. Para este caso, se va a crear  uno llamado readonly para este tipo de operaciones.

Primero generamos el hash de contraseña del usuario bind DN:

root@orion:/home/administrador# slappasswd

El hash obtenido es el siguiente: {SSHA}1eAQhGMXpC0Swd+PkKMlXPLTWYdSzyZJ. Este hash hay que copiarlo y pegarlo en un nuevo archivo llamado readonly-user.ldif

root@orion:/home/administrador# nano /etc/ssl/openldap/readonly-user.ldif

En este archivo copiaremos lo siguiente:

dn: cn=readonly,ou=usuarios,dc=ciber,dc=local
objectClass: organizationalRole
objectClass: simpleSecurityObject
cn: readonly
userPassword: {SSHA}1eAQhGMXpC0Swd+PkKMlXPLTWYdSzyZJ

description: Usuario Bind DN para operaciones en LDAP.

Por último, se añade este usuario a la base de datos de LDAP:

root@orion:/home/administrador# ldapadd -Y EXTERNAL -H ldapi:/// -f /etc/ssl/openldap/readonly-user.ldif
