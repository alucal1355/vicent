8. Gestión de accesos. Sistemas NAC

1. Introducción

Actualmente muchas empresas están utilizando el teletrabajo como una de las soluciones para permitir que varias personas operen desde el exterior de la empresa, como si estuvieran en ella, es con una red privada virtual o VPN. El sistema operativo en red Windows Server 2022 incluye este rol dentro del listado de funciones que pueden añadirse como servicio en el sistema. Como solución, se puede realizar una implementación simple de VPN utilizando un único servidor, pero como medidad de sguridad, se puede completar el uso de esta tecnología con un servidor RADIUS (Remote Authentication Dial-In User Service).

VPN es una tecnología de transporte de datos a la que se complementa de tecnologías de cifrado y autenticación. Con RADIUS realizamos esta última función.

Un caso práctico de autenticación RADIUS, sería la autenticación a una red wifi con WPA2-ENTERPRISE en el cual no necesitamos una clave secreta compartida para acceder a ella. En lugar de una contraseña, utilizaríamos un usuario para autenticar.

2. ¿Qué son los sistemas NAC?

El sistema NAC, o Network Access Control, es la tecnología que nos permite controlar dispositivos para que pueden acceder a la red, permitiendo establecer políticas de gestión de los dispositivos.

Con un sistema NAC se impide el acceso a la red a dispositivos que no cumplen los requisitos, este los mueve a un área en cuarentena o permite el acceso restringido a los recursos.

3. Importancia de tener una solución NAC

El control de acceso a los dispositivos de la red de la empresa es algo muy importante, ya sea cuando se accede desde el exterior o desde dentro de la red. 

Es común que los empleados utilicen sus propios dispositivos para acceder a los recursos de la empresa. A esto se le llama fenómeno BYOD, y hace necesario el uso de tecnologías NAC, con las que se consigue autenticar al usuario y su dispositivo.

4. Capacidades de las soluciones NAC

Las soluciones de NAC controlan el acceso a las redes a través de los siguientes puntos:

  • Administración de la políticas.  Aplica políticas de todo tipo sin necesidad del uso de herramientas de terceros.
  • Creación de perfiles y visibilidad.  Crea perfiles de los usuarios según el perfil que estos tengan y de sus dispositivos.
  • Acceso a la red de usuarios temporales.  Mediante el uso de un portal de autoservicio incluimos el registro, autenticación y un portal de autoservicio para los usuarios de temporales y así poder administrarlos.
  • Comprobación del estado de seguridad.  Evalúa si se están cumpliendo las políticas de seguridad por el tipo de usuario, dispositivo y sistema operativo.
    Respuesta ante incidentes. Mediante el uso de políticas de seguridad que bloqueen, aíslen y reparen las máquinas con amenazas sin que el administrador realice ninguna acción.  De esta forma reduciremos las posibles amenazas en la red.
  • Integración bidireccional. Se integra a otras soluciones de red y seguridad a través de una API REST.

5. Casos de uso

  • Control de proveedores que trabajan en una red corporativa.
  • Control de oficinas remotas.
  • Control de redes en infraestructuras críticas: redes industriales, plantas nucleares, etcétera.
  • Control de les redes corporativas.
  • Gestión de BYOD en cualquier red corporativa; un ejemplo claro son los campus universitarios, o los centros educativos que cuentan con centenares o millares de alumnos/as.
